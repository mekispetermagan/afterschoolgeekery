Click Counter
What is this project about?

This is a fun first project for learning mobile app development. The app counts how many times the user presses a button in 30 seconds. After the project work, kids can compete with each other in who has the fastest hands.

Step 3: Second design with timer

It would be nice to also measure time, so that kids can compete with each other. Discuss how you used to do this in Scratch. There it was supersimple: besides the built-in timer, the "wait" block also did the job of measuring time. In App Inventor, you don't have these. You need to add a clock component to your projects. The clock's ticking will be the event to assign timed code to.

Switch back to the Designer interface, and look for a clock component. You will find it in the "Sensing" category:

The clock has a "TimerIinterval" property. It tells how fast the clock is ticking. You can set it faster or slower, like a metronome. The number is in milliseconds; 1000 means a one second interval, which is exactly what you need.

The clock is a hidden, invisible component. It's ticking in the background, triggering events for other components. But you will also need to display the time. Discuss with the kids how to do that. Another label will be a nice choice. A lot can be done with just buttons and labels. Discuss how many seconds you want to give the user to click, and set the label's text to that number. Avoid very large numbers to avoid spending too much time on testing. Like before, use the Properties panel to stylize the label. You may even use different colors for the clicks and the time labels:

Step 5: Start at first click

If you still have time left: It would be nice for the countdown to start at the first click. This part needs some thinking, so give yourselves some time clarifying how to achieve it.

One possible idea is to partition the game into three states:

  1. before the game starts: ready state
  2. while the game is going on: play state
  3. when the game is over: done state
Of course, the naming of these states doesn't matter; you can even use the numbers 1, 2, and 3 for them. What matters is that you need to keep track of them. Probably the simplest way to do this is using a variable. You can initialize it to "ready":

Follow-ups

If you still have time left, you can add many more features to your click counter:

  • A reset button that sets back click count to zero, the time to 30, and game state to "ready".
  • A high score variable, with its value displayed on another label.
  • Yet another label that displays the maximum number of clicks per second. Can anyone in the group go beyond 10? :)
