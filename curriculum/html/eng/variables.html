Variables Guide

Introduction
What is this guide about?

In this guide you will learn how to use variables in programming through two simple projects. Variables are tools that help the computer remember data that will be needed later in your program. This can be text data (for example, a user name) or a number (for example, your score in a game).

You can think of variables as sticky notes you can use to write down things you want to remember. It is a good idea to give the sticky note a title, so you know which note to use to look up the thing you wrote on it: for example, "username", "score", etc.
Later, when the value changes (a user name is changed or new points are earned), the data is rewritten on the sticky note. Whether the user collected 5 or 500 points, we can find the exact value on the "score" sticker.

Variables work in a similar fashion. A variable is a value, a piece of data (number or text) that has a name. The variable name stands for the sticky note's title; and the value stands for whatever is written under the title on the note. Just as a sticky note's content, a variable's value can be changed while your project is running, which is why it's called a "variable".

The first project is very simple and short; you will add a variable to an existing game, improving the game experience. This project will let you learn how to create a variable, and how to use Scratch's variable blocks. See the end result here:

(About the other options:

  • Variables that are "for this sprite only" will be useful when you duplicate or clone sprites, and each of them has a different value; like the kids in your course all have their own ages; or like every character in a fighting game have their own health. This type of variable is called a local variable, as opposed to the global variables that we use in this guide. Local variables are discussed in project 3 of the Cloning guide.
  • Cloud variables are only used in online Scratch, and they are only available for Scratchers who have shared a few projects already, and have engaged themselves in the social part of Scratch, liking, commenting and following others' work. The values of these variables stored in Scratch's servers, and they are the same for everyone who opens a Scratch project, whether in Vietnam or in Greenland. These are typically used for saving world-wide high scores for a game, or to counter how many times a Scratch project has been opened.)
