Quiz project
What is this project about?

It's a relatively simple project that can be done in one session, and gives you a lot of scope for creativity. As it uses variables and a list, this project is not for beginners. You will be making a multiple-choice quiz, which everyone can fill in with questions taken from their own favorite topic. You can see the final result here:

Step 1: Topic selection

Before you sit down at the machines, discuss who will be taking a quiz on which topic. This could even serve as an icebreaker game at the beginning of the session. To make brainstorming easier, you start with your own topic and ask a question straight away. For example:

Before you start coding, it's a good idea to plan what the quiz will look like (in short, design the user interface) so that kids can see what you'll be writing the code for. The simplest way to go is having both the questions and the answers written on the background, and the only sprites will be the three buttons with the letters A, B, C. Step 3: Code of the stage

After all the planning, now comes the well-deserved coding! Start with the background here too, because it's the easiest. The background's job is to always show the question you're working on; the first, the second, the third, and so on. So for Scratch to know where you are, you need a variable that counts the questions; similar to how in other games you count points, lives, levels, etc. with a variable. Create a new variable called "question". At the start of the program, the value of the variable should be 1, since we start with the first question; and from then on, whatever the value of this variable is, the background should always be adjusted: first question => first background, second question => second background, and so on, always according to the value of the variable.

Really, how will Scratch know which answer is correct? Well, how will the TV presenter who hosts a quiz show know? He's holding a sheet with a list of correct answers that only he can see, the player can't. We will do the same thing, making a list of the correct answers.

If this is the first time you are using a list, discuss in detail what it is: a variable in which you can enter not just one piece of data, but several ones at once. It adds a lot to the game experience if the correct and incorrect answers are accompanied by sounds. If you haven't worked with sounds before, this is a great opportunity to try it out. To avoid having to insert sounds and new code in three places, we give the background the task of playing sounds. Make sure that the background is selected, and to avoid complications later, everyone should lower the volume on their laptops so that they can still hear the sounds, but not disturb others.

Everyone can search for the sounds that best suit them. If you don't feel like searching, "Cheer" can be used for the right answer, and "Chee Chee" for the wrong one.
