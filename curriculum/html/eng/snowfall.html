Snowfall
What is this project about?

This is a snowfall animation where the level of snow keeps raising, and eventually snow covers the whole Scratch stage. It is a "clones falling" project like "Rain" in the Cloning guide, or the Flower Shower miniproject, but with a few twists. The idea behind using the same code pattern in multiple projects is that repetition in a different context and setting will help kids master this technique, which consists of too many steps to memorize at first, but will be very useful in creating their own animations and games.

Use this project with kids who already have some experience with Scratch. Although you can introduce cloning with this project, it is a good idea to schedule it after one of the simpler cloning projects like "Flower Shower" or the Dino game, so that the cloning part goes more fluently, and enough time remains for the rest. Here in this guide, the discussion of the cloning part will be brief (few words, many pictures); see the "Rain" and "Flower Shower" projects mentioned above for more detailed guidance on the technique of teaching this technique.

Preparations

Create a new project, and say goodbye to Scratchie. Click on the cat head icon at the bottom right to get to the sprite library, and select the sprite called "Snowflake". Click on the image icon next to the cat head to select a nice outdoor background. (Of course, you can also draw your background or snowflake if you are confident that it will not consume too much time.)

The costume of the snowflake sprite is a bit bluish. If you think snow should be white, this can be done in a few moves. Open the Costumes tab and select the whole snowflake (with mouse or Ctrl A). Set the fill color to white (saturation to zero, brightness to 100) and the outline color to light gray (saturation to zero, brightness to any number above 50).

Finally, set the size of the sprite under the stage to a very small value, so that there will be room for thousands of snowflakes:

Step 1: A single snowflake falling

As usual, let's start with moving a single sprite before setting up a clone army. If the group is already familiar with coordinates, ask them about the coordinates of the edges of the stage. If they don't remember, use the background with coordinates to refresh their memory. If they are too small for coordinates, use the place-and-pick method:

  1. Place the sprite at the top of the stage where it starts from.
  2. Pick a "go to" block with the current coordinates.
  3. Place the sprite at the bottom where it arrives.
  4. Pick a "glide" block with the new coordinates.
  5. Nest the two blue blocks in a "forever" block to make the movement continuous.
Restart the project to see if the sprite falls as intended.

Now randomize the horizontal position in a similar way:

  1. Pick a "random" block and place it in the "x" slot of the "go to" block.
  2. Place the sprite at the left edge of the stage, and read its x coordinate.
  3. Place it at the right edge, and read the x coordinate again.
  4. Write the two values in the "random" block.
  5. Put an "x position" block in the "x" slot of the "glide" block to secure that the x coordinate doesn't change while gliding.
  6. Set the duration of falling in the "glide" block.
Restart the project a few times to see if the code works.

Step 2: Cloneflakes falling

If you have done cloning before, this will be easy. If you haven't, slow down and make sure kids get what they are doing. Find the three cloning blocks in the Control group. Put the movement blocks aside, they will belong to the clones. Now place the three clone blocks: "create clone" is nested in "forever"; the event block goes over the movement blocks; and "delete clone" under them.

If you are running out of time, you can close project work now with success. But if you still have some time and energy left, the next few steps will be new and fascinating.

Step 3: Keeping the fallen snowflakes

It would be nice not to delete the fallen snowflakes, so that they create a nice sheet of snow at the bottom. The problem is that Scratch only allows for 300 clones, and you need those for the new ones to fall. There is a solution hidden deep inside the Pen extension of Scratch. It's not a problem if you haven't used Pen blocks before; you will need only two of them, and they are rather self-explanatory.

Click on the extension icon at the bottom left, and select "Pen". You get a bunch of new blocks:

Step 4: Raising sheet of snow

You could make the animation even more amazing by letting the fallen snowflakes accumulate, simulating the raising sheet of snow created by the snowfall. The idea is good, but to implement it in the project, you have to find a way to translate it to Scratch. What does the raising sheet mean for the falling snowflakes? Simply that they will fall, or in Scratch terms, glide shorter and shorter distances. More precisely, the number in the "y" field of the "glide" block will keep increasing, from -180 (or whatever coordinate you got for the bottom) towards zero (which is the middle of the stage), and further towards 180 (or the number you got for the top).

There are many ways to do this. You can introduce a variable to measure time (like in the "Click counter" project of the Variables guide). Or you can introduce a variable for the current level of the snow, starting from -180 and gradually increasing.. You can find instructions on how to do these in the Variables guide in case kids want to go this way. But there is an even simpler way. The Sensing block category has a "timer" block that returns with great precision the number of seconds elapsed since the starting of the project. You can just add that to whatever number you had in the "y" field of the "glide" block originally:

Step 5: Swinging snowflakes

If you still have time left, you can make the snowflakes swing left and right in the air while falling. This, again, requires a little bit of ideation and analytical thinking; but this is exactly what we are learning! :) The snowflakes should be falling and swinging at the same time. The falling part is done. What is swinging? How does a swing move? It turns forward around its axis (the rod it hangs from), then turns backward, and so on. Can we phrase it in Scratch language?
