Simple quiz project
What is this project about?

Quiz projects are very popular among kids, but they can be quite challenging, because the logic of a quiz is not as simple as that of an animation or a shooting videogame. The intention of this project is making quiz accessible to kids at less advanced level of scratch skills.

The main differences between this and the more advanced quiz project:

  • Instead of multiple choice questions, this one has "yes-now" questions, thus simplifying the creative process, reducing the amount of text to type, and the code to put together.
  • In the advancd quiz project solutions are stored in a list, which is an advanced data type, and can be difficult to grasp for some kids. In this simplified version solutions are stored in background names, which can be accessed with a much simpler code.

All in all, it is a good idea to start with this simple quiz project, and move on to the more advanced one a few weeks later.

Step 1: Topic selection

Before you sit down at the computers, let each kid choose a topic their quiz. This can be your icebreaker game for the session. To make brainstorming easier, you start with your own topic and show them a true and a false sentence straight away. For example if your topic is geography:

--and so on. After each pair of sentences, try to guess collectively which is true and which is false.

Step 2: interface design

Probabaly the easiest way to put together a quiz interface is writing the questions on different backgrounds, and using button sprites for answer options. To get the buttons, click on the cat head icon under the stage, and choose any of the button sprites:

Of course, you can also draw your own buttons.

Use the text tool to indicate the functions of the buttons, and if you feel like, customize them.

Besides the two buttons, the quiz interface will contain the current sentences. Write your first few sentences on backgrounds, and decorate them as you like. Place the buttons below the sentence in the stage.

Would you like to know how to make your letters bold? :) It's tricky and fun!

The names of these backgrounds matter; the program will use background names to tell which sentence is true and which isn't. Name the true ones "true", and the false ones "false" (or whatever you like; just make sure you know which is which, and that their first letters differ like "t" and "f"; that is what we will use.) The editor will automatically rename the second true one as "true2", the third as "true3" etc.

Now that you have the design and at least some of the content, you can start coding.

Step 3: Showing and switching sentences

The quiz starts with the first sentence; then it moves to the second, the third, and so on. But how does Scratch know at any point of the quiz which is the current queston? It looks like we have to keep track of the number of the current question; a number that keeps changing during the gameplay, much like score or speed or ammunition keep changing in videogames. The tool that we use for this purpose is called a variable; a piece of storage that holds a value that keeps changing (hence the name variable), and makes it accessible by a name: "score", "speed", or, in our case, "question". You can learn more about this in the Variables guide. If you introduce variables in this session, take your time, and make sure you discuss every step with the kids.

Scroll down to the Variable group in the blocks panel at the left, and create a variable called "question". Both the stage and the buttons have business with this variable; let's see them one at a time.

Let's start with the stage. What is the initial value of the variable "question"? This sounds very abstract, so let's put it another way: Which question does a quiz start with? This is the kind of question that a kid is ready to answer: the first, of course, question number one. So, at the beginning of the project "question" should be set to 1.
