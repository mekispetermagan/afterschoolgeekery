Bouncing ball
What is this project about?

This short animation project can be put together in no time, and it can be a good choice for the first session, even with small kids.

A colorful ball will be bouncing back and forth in the stage, changing color from time to time. Optionally, the ball will also draw a colorful line. The project is flexible; if there's no time for the colors, you can still finish it with a feeling of success.

Preparations

If this is the first session for the kids, watch this animation together, and recreate the code together to get a first experience of how things work in Scratch.

Start a new project, say goodbye to Scratchie cat, and select the sprite called "Ball". This choice is not necessary; if some of the kids want another sprite, let them choose that.

Step 1: Moving the ball

Now let's move the ball. We will do this the simplest way, using the "move" block, as you saw in the second projekt of the Motion guide. Let's start with a single movement.

Ask the kids what block starts a code, some of them will surely remember that it is the "green flag". Also ask them what block moves a sprite. If they have just seen the animation, they will mention the "move" block. If this is not the first session, and they have already created a few projects, and they mention "glide", "go to" or "change x", just tell them to find a simpler motion block, and they will find it.

You can leave the number inside "move" as it is, or let the kids set it lower or higher, as they like.

Start the project for the first time. The ball moves a bit every time you click on the green flag, but then it stops. Ask the kids to make the movement continuous, they will know what to do:

Start the project and see how the sprite starts to move towards the right edge, where it gets stuck. To get the sprite back to its original position, they can just grab it with the mouse. Sometimes, when a kid puts a huge number in the "move" block (they love experimenting with large numbers, and this is a good thing), the sprite gets completely lost from the stage; then they can retrieve it by typing zero in the x and y fields under the stage.

Step 2: Make the ball bounce

Now ask the kids if they can find a motion block that tells a sprite what to do when it reaches the edge of the stage. They will find the "if on edge, bounce" block in no time. To make the movement of the ball more interesting, it's time to change its direction. Ask the kids if they can find the "direction" field under the stage. When they found it, tell them to click into it, and explore how it works. Every time they change the direction of the ball with the pointer, the ball will change its movement.

Now the sprite behaves just as a good ball should do; it bounces in various directions, traversing the whole stage.

If you have reached this far, you can stop your project work with success; nice job on behalf of both you and the kids. But if you have some time left, you can make the animation more spectacular in two more steps.

Step 3: Costume change

Start a new script, placing another "green flag" block in the script area. This is how Scratch works; instead of building one tall tower of blocks, you build a group of small towers, each of them serving different purposes.

Check out the costumes of the sprite clicking on the "Costumes" tab. Even if some of the kids chose another sprite, it will probably have more than one costume. Now click on the "Code" tab, and ask the kids to find a block that switches to the next costume, telling them that it will have a different color than the blocks that you used so far.
