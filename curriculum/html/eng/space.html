Space game
What is this project about?

This is a typical arcade style "shoot and dodge" game. It is slightly more difficult than the Dino game, so it is only recommended when kids have already had a few sessions of experience with Scratch. Here is the end result:

In this guide we will assume that the kids are already familiar with the basics of movement and cloning. If they are not, this project may be a bit too complex to introduce it.

Preparations

After showing the project to the group, let them open this initial version. This has all the sprites and graphics, and about half the code.

In particular, 8 scripts in 3 sprites are incomplete. The project work will be the completion of the code. The incomplete parts are easy to spot:

By the end of the project work, these will all be filled with code:

Start the game with the green flag. It passes the title, but then the spaceship just stays idle, and nothing happens. Discuss with the kids what features are missing, and write a list of the tasks. You will come up by something like this:

  • moving the spaceship
  • enemy ships coming
  • game over when touching enemy or its bullet
  • shooting
  • enemy ships are destroyed when touching bullet
  • enemy ships shooting
  • displaying score
Out of these, the last two are pre-coded and included in the project, becuse they are slightly more complicated than the previous ones. So you can cross these. Coding the rest will be your project work for the session.

Make shooting optional. It's very cool if you have enough time for it; but the game is playable without it just as well. So your final list of tasks will look something like this:

  • Must have:
    • moving the spaceship
    • enemy ships coming
    • game over when touching enemy or its bullet
  • Nice to have:
    • shooting
    • enemy ships are destroyed when touching bullet
  • Already in the code:
    • enemy ships shooting
    • displaying score
This distinction between "must have" and "nice to have" tasks is very useful in project management.

Step 1: Moving the spaceship

The spaceship moves when the left or right arrow keys are pressed (of course, other keys are just as good, like A and D for gamers). You have probably done this before. Remind the kids that there are two ways to sense keypress, and ask them which one is better for game control, and clarify that the light blue sensing block is clearly better when the player holds down a key during gameplay. (This is discussed in detail at the end of the Motion guide). The hexagonal block fits in a "wait until" one (or "if-then", it doesn't matter in this case), so the first script will be straightforward:

It is a good idea to let the kids put together the code for the other direction themselves. If they get stuck, remind them of negative numbers with a guiding question: "If right is plus five, what can left be?"

Restart the project to check if the spaceship moves properly.

Step 3: Touch sensing and game over

In fact the script is almost complete; all you have to do is fill the hexagonal slot with sensing blocks. But there is a twist. The game ends when the spaceship touches either an enemy ship or an enemy bullet. So we need two sensing blocks, but there is only one spot:

This is a kind of problem that is puzzling at first, but becomes almost trivial once you have solved it once. Remember that Scratch is a language, so whatever you want from the game, if you can phrase it properly in normal language, you can also translate it. We said "either-or" when phrasing the task.
